Project

First-Year Project

As part of our first-year multidisciplinary project, we were divided into teams to carry out a practical task that combined three key subjects: electronics, C programming, and system analysis. The main objective was to design an interactive dice game. The rules were simple: each player rolled two dice and added the results. If the total exceeded 55, the player lost the game. The game continued until someone won. From the electronics side, we used 12 LEDs to show the values rolled by the dice. For the programming part, we had to write a program in C that followed the game’s rules. We programmed the dice rolling logic and made sure the scores updated correctly after each turn. At the end, we linked the electronic part to our code so that the whole setup could run smoothly as one system. Even though we faced a few challenges, this project helped us become more independent, learn how to look for solutions by ourselves, manage our time better, and most importantly, how to work effectively in a team.

Second-Year Project

For our second-year multidisciplinary project, we were divided into several teams based on our individual skills and knowledge. The project's goal was to sort balls of two different colors so that each one would end up in the correct container. To achieve this, we placed several balls inside a pipe that contains two main components, an ultrasonic proximity sensor and a servo motor. The ultrasonic sensor detected when a ball was present. Once detected, the servo motor quickly rotated from 90° to 0°, then returned to its original position. This mechanism allowed us to release the balls one by one, spaced a few seconds apart. After being released, the ball dropped into the detection zone of a HuskyLens camera. The camera recognized the ball's color and displayed the result on an LCD screen to inform the user. If the ball was pink, the stepper motor moved forward by 100 steps, paused for one second, and then moved back. If the ball was yellow, the movement was reversed. The ball was then guided through a pipe into the appropriate container. However, if the correct container was full or the ball was neither pink nor yellow, it was redirected to a mixed container. This project was an excellent opportunity to work with color recognition, real-time actuation, and team-based decision-making, all while combining electronics, programming, and automation in a practical scenario.

Third-Year Project

As part of my third-year project, we were required to define and write our design brief. I decided to create an autonomous system capable of regulating the temperature inside a small box based on a predefined threshold. A temperature sensor continuously monitors the internal temperature. If it exceeds the threshold, a fan turns on to cool the box. If it drops below the limit, a heating lamp is activated to raise the temperature. To carry out this project, I used an ESP8266 microcontroller to manage both the data processing and the communication. It sends real-time information to a mobile application. On the software side, I developed a mobile app using React Native with Expo Go to display the data sent from the ESP8266. For example, the app shows the current temperature of the box in real time. The app includes several screens: a homepage, a dashboard showing the live temperature, and a history page displaying all recorded values. This historical view also consists of a real-time graph for better data interpretation. This project helped me combine embedded systems, mobile development, and real-time monitoring skills, which I’m eager to keep developing in my future career.
